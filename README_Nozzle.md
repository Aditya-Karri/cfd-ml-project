# Physics-Informed Machine Learning Surrogate for Supersonic Nozzle Flow

## 1. Problem Statement
While Computational Fluid Dynamics (CFD) is the industry standard for high-fidelity compressible aerodynamic analysis, it is notoriously computationally expensive. A single simulation of a 2D axisymmetric converging-diverging (CD) nozzle in ANSYS Fluent takes approximately 10 minutes to converge.

The objective of this project was to build a Machine Learning surrogate model capable of predicting both the macroscopic performance (Thrust) and microscopic fluid dynamics (the 1D centerline Mach distribution) in sub-milliseconds, using solely the Nozzle Pressure Ratio (NPR) as an input.

## 2. Dataset Generation and CFD Setup
To train the surrogate, a high-fidelity dataset was generated by automating ANSYS Fluent using a custom Python wrapper (subprocess + Fluent TUI journal files).

* **CFD Solver:** ANSYS Fluent (2D Axisymmetric, Density-Based Steady Solver)
* **Fluid Properties:** Compressible Air (Ideal Gas Law), Energy Equation Enabled.
* **Turbulence Model:** $k-\omega$ SST (Shear Stress Transport).
* **Run Matrix:** 18 fully automated, independent CFD runs simulating an NPR sweep from 2.0 to 18.0. The sampling was adaptively clustered in the highly dynamic over-expanded regime (NPR 2.0 to 6.0) to capture complex internal shock structures.
* **Extracted Data:** Converged Thrust, wall pressure, 2D field data, and 1D centerline Mach distributions (interpolated to a uniform 400-point spatial grid).

## 3. The Baseline Challenge: The Smoothing Effect
Initially, a standard Convolutional Neural Network (CNN) was trained to map NPR directly to the Mach distribution.

The Engineering Critique: The baseline CNN struggled significantly at lower NPRs where strong normal shockwaves occur inside the nozzle. Because standard CNNs trained with Mean Squared Error (MSE) inherently struggle with sharp, violent discontinuities, the network hedged its uncertainty by predicting highly jagged, unphysical, and "smoothed" Mach distributions. The baseline model simply did not know that the spatial curve it was drawing had to mathematically relate to the macroscopic thrust being produced.

## 4. Machine Learning Architecture (PINN)
To enforce physical realism, the architecture was upgraded to a Multi-Task Physics-Informed Neural Network (PINN) with a shared latent core.

### Multi-Task PINN Architecture Diagram
```text
INPUT        LATENT CORE               BRANCHES (Multi-Task)          OUTPUTS

                                 ┌───> [Dense Layers] ──────────────> Macroscopic Thrust
                                 │     (Physics Constraint Branch)
[NPR] ──> [Dense 128 (ReLU)] ────┤
          [Dense 128 (ReLU)]     │     (Data Matching Branch)
                                 └───> [Reshape] ──> [Conv1D] ──────> 1D Mach Curve (400 pts)
```

The Physics Prior:
A physics constraint was introduced via a custom loss function weighting. A heavier multiplier ($\lambda = 2.0$) was applied to the Thrust prediction branch compared to the Mach branch ($\lambda = 1.0$).By forcing the optimizer to prioritize the macroscopic momentum balance, the network was effectively forced to align the spatial Mach curve with the physical thrust constraint, completely eliminating the erratic jaggedness of the baseline CNN.

## 5. Performance: Speed vs. Accuracy Tradeoff
The PINN architecture successfully stabilized the outputs and reduced microscopic fluid dynamic errors by 32%, resulting in a surrogate capable of sub-millisecond inference times.

| Metric                         | ANSYS CFD (Ground Truth) | PINN Surrogate 
| :---                           | :---                     | :---
| **Compute Time / Case** | ~567 seconds             | 0.018 seconds
| **Computational Speedup** | Baseline                 | **30,907x Faster**
| **Thrust Error** | Baseline                 | 29.4% 
| **Mach Curve RMSE** | Baseline                 | 0.1809 (32% Improvement over standard CNN)

**Failure Analysis (The Normal Shockwave Problem):**
While the PINN mathematically smoothed the curves and improved RMSE by 32%, the model still struggles to predict the exact, instantaneous spatial location of the discontinuous normal shockwave. This is due to data starvation (only 18 highly diverse CFD samples) and the mathematical limitations of standard gradient-descent optimizers when handling absolute step-functions in fluid regimes. Future iterations will require gradient-aware loss functions that explicitly reward steep spatial gradients.

## 6. How to Reproduce Results
The repository isolates production scripts from intermediate R&D experiments.
1. **Create and activate the Conda environment:**
```bash
 conda env create -f environment.yml
 conda activate cfdml
```
2. **Run the Airfoil Inference Demo for any Angle of Attack (e.g., 7.5 degrees):**
```bash
 python predict_airfoil_demo.py --aoa 7.5
```
3. **Review R&D / Baseline Models:** Navigate to experiments/nozzle_surrogate/ to view the Python scripts detailing the baseline CNNs and failure analysis.
4. **Run Production Training:** Navigate to experiments/nozzle_surrogate/physics_aware_nozzle_surrogate.py to view or execute the custom multi-task training loop with the $\lambda=2.0$ physics constraint.
